# PVZTI 题库生成 Prompt

请你担任一位兼具文学感、游戏感与心理测评设计能力的题库作者，为一个名为 **PVZTI** 的植物人格测评网站创作题目文案。

## 任务目标

生成一份 **严格合法的 JSON**，用于网站的 20 题人格测评配置。

这份测评借鉴 MBTI 的体验节奏，但人格维度不是 MBTI，而是 6 种植物：

1. `peashooter` / `豌豆射手`
   特征：主动、直接、推进快、喜欢先手打开局面、重视效率和即时反馈。
2. `sunflower` / `向日葵`
   特征：供能、支持、协调、重视长期关系与持续性、擅长稳定氛围。
3. `wallnut` / `坚果墙`
   特征：守护、抗压、边界感强、可靠、愿意承担风险和责任。
4. `potatoMine` / `土豆地雷`
   特征：低调、观察、耐心、等待时机、关键点爆发、擅长隐藏判断。
5. `cabbagePult` / `卷心菜投手`
   特征：抽离、统筹、结构感、全局视角、擅长从系统层面思考。
6. `cherryBomb` / `樱桃炸弹`
   特征：爆发、果断、关键时刻愿意破局、决策强、情绪能量高。

## 创作要求

1. 总共输出 **20 道题**。
2. 每题必须有 **4 个选项**。
3. 每题都要是 **场景化、可代入、有画面感** 的中文表达，不要写成生硬问卷口吻。
4. 题目风格要轻巧、有一点幽默感，但不能油腻，整体像是“游戏人格测试 + 文学化表达”。
5. 每题选项必须体现 **真实取舍**，不要出现一看就“更正确”或“更成熟”的答案。
6. 所有维度要尽量 **均衡出现**，不要让某两个植物明显被偏爱。
7. 选项分值使用相对克制的方式：
   - 主维度通常给 `3`
   - 次维度通常给 `1`
   - 允许偶尔使用 `2 + 2`
   - 不要出现负分
8. 每个选项都必须至少命中一个植物维度。
9. 选项 `tone` 字段要用简短中文短语概括气质，例如：`"主动型、进攻型"`、`"结构型、统筹型"`。
10. 输出必须是 **纯 JSON**，不要加解释、不要加 Markdown 代码块、不要加注释。

## JSON 结构要求

输出结构必须严格符合下面格式：

```json
{
  "version": "2026-04-14-draft",
  "totalQuestions": 20,
  "questions": [
    {
      "id": "q01",
      "title": "启动方式",
      "prompt": "一个全新的项目突然落到你面前，你第一反应更像哪一种？",
      "options": [
        {
          "id": "q01-a",
          "label": "先把资源、人手和情绪安顿好，再稳稳起步。",
          "tone": "蓄能型、照料型",
          "scores": { "sunflower": 3, "wallnut": 1 }
        },
        {
          "id": "q01-b",
          "label": "先冲出去打第一枪，边做边打开局面。",
          "tone": "主动型、进攻型",
          "scores": { "peashooter": 3, "cherryBomb": 1 }
        },
        {
          "id": "q01-c",
          "label": "先埋好伏笔，等关键节点再一击命中。",
          "tone": "潜伏型、时机型",
          "scores": { "potatoMine": 3, "cabbagePult": 1 }
        },
        {
          "id": "q01-d",
          "label": "先站远一点看全局，梳理路线后再推进。",
          "tone": "统筹型、结构型",
          "scores": { "cabbagePult": 3, "sunflower": 1 }
        }
      ]
    }
  ]
}
```

## 额外约束

1. `id` 必须连续命名：
   - 题目 `q01` 到 `q20`
   - 选项 `q01-a` 到 `q20-d`
2. `title` 要精炼，2 到 6 个汉字为宜。
3. `prompt` 要比 `title` 更完整，读起来像一句真正的问题。
4. `scores` 的 key 只能来自这 6 个值：
   - `peashooter`
   - `sunflower`
   - `wallnut`
   - `potatoMine`
   - `cabbagePult`
   - `cherryBomb`
5. 不要输出空字符串。
6. 不要输出重复题意。
7. 不要把题目写得像职场问卷，生活、关系、决策、压力、合作、独处、机会判断等场景都可以覆盖。

## 风格目标

你写出来的题应该让用户感觉：

- “有点像在玩一个世界观完整的小测试”
- “每个选项都像不同植物会说的话”
- “不像标准心理量表，但也不是随便胡编”

现在开始输出最终 JSON，不要输出任何解释文字。
